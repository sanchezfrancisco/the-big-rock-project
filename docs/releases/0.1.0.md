# Release Notes

## Release

- Version: `0.1.0`
- Date: `2026-05-05`
- Type: `MINOR`

## Summary

`0.1.0` delivers the first production-ready local release of Semantic Code Navigator with a complete operator workflow across CLI, API, and Web UI. The release includes local-first indexing and retrieval, explainability, evaluation/tuning loops, embedding cache operations, runtime backend selection, and release governance documentation.

## Highlights

- End-to-end local workflow: `index -> ask/explain -> eval -> tune -> gate`.
- Runtime embedding backend selection (`local` / `openai`) from UI/API.
- Persistent embedding cache with prune and telemetry.
- Dynamic Ask suggestions with `Tab` accept in Web UI.
- Observability event stream for operations and quality tracking.

## CLI Changes

- Added:
  - `scn explain`
  - `scn eval`
  - `scn tune`
  - `scn cache-stats`
  - `scn cache-prune --max-entries N`
  - `scn doctor`
- Changed:
  - `scn status` now includes cache details.
  - `scn eval` supports `--report-out` and `--min-recall`.
- Deprecated:
  - None.
- Removed:
  - None.

## API Changes

- Added endpoints:
  - `POST /explain`
  - `POST /eval`
  - `POST /tune`
  - `GET /cache/stats`
  - `POST /cache/prune`
  - `GET /metrics/recent`
  - `GET /suggest`
- Added runtime override fields in supported request bodies:
  - `embedding_backend`
  - `hybrid_keyword_weight`
  - `embedding_cache_path`
- Breaking changes:
  - None in this release line.

## UI Changes

- Added operational console covering:
  - Indexing, Ask, Explain, Eval, Tune, Cache Stats/Prune, Status.
- Added runtime controls:
  - Backend selector (`local`, `openai`)
  - Hybrid weight input
- Added Ask autocompletion behavior:
  - dynamic suggestions while typing
  - `Tab` to accept suggestion
- Added observability panel:
  - recent metrics events (`/metrics/recent`)

## Quality and Performance

- Eval workflow enabled with report export and threshold gates.
- Tuning workflow enabled for hybrid weight search.
- Embedding cache introduced to reduce repeated embedding cost in reindex loops.
- Baseline note:
  - Local hash embeddings may produce low semantic recall in complex queries.
  - OpenAI backend is available for higher quality retrieval.

## Security and Privacy

- Local-first backend remains default (`local`).
- Secret redaction is applied during discovery/indexing.
- `.scnignore` available for path-level exclusion.
- OpenAI backend remains explicit opt-in and environment-key based.

## Migration Notes

- Environment/config additions:
  - `SCN_EMBEDDING_CACHE_PATH`
  - `SCN_OPENAI_EMBED_MODEL`
- Optional dependency group:
  - `pip install -e ".[providers]"`
- Required manual actions for `openai`:
  - set `OPENAI_API_KEY`
  - select backend in UI/API or env

## Known Issues

- Retrieval quality with `local` hash embeddings is limited for broad semantic questions.
- Suggest endpoint depends on index quality and can return empty suggestions on low-signal queries.
