# Agile Learning System

## Operating Rules

- Quarterly OKRs measure shipped AI capability, not passive study.
- Weekly Kanban WIP limit: 2 active learning tasks.
- Buffer rule: reserve 20% of weekly hours for blockers, review, or deeper practice.
- JIT learning: if math or theory blocks progress for more than 3 days, treat it as a black box and continue. Reopen it during buffer time.

## First Two Months OKRs

### Objective 1: Integrate AI into a real software workflow

- KR1: Index at least one complete local Python repository.
- KR2: Answer technical questions with citations to files and lines.
- KR3: Keep monthly AI/API cost under $20.

### Objective 2: Build sustainable engineering habits

- KR1: Keep WIP at 2 or fewer tasks.
- KR2: Maintain a weekly blocker/decision log.
- KR3: Produce a reproducible CLI demo by the end of month 2.

## Weekly Demo Checklist

- Run `scn index <repo>`.
- Run `scn ask "<technical question>" --top-k 3`.
- Verify every answer against its citations.
- Log one improvement for retrieval quality, security, or developer experience.
